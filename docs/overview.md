# 整体概览

## 项目定位

该项目旨在将 `Jira` 的项目管理能力与 `Codex CLI` 的分析、设计、任务拆解和执行能力结合，构建一个面向研发流程的 AI 半自动化协同体系。

核心定位如下：

- `Jira` 负责流程控制、任务记录、排期协作、审计留痕
- `Codex CLI` 负责需求理解、故事定义、任务拆解、技术设计、代码执行、结果回写
- `Jira Skill` 负责连接 Jira API 与结构化数据写回
- `Orchestrator` 负责编排触发、调用、审批和执行流程

## 核心目标

- 提升 AI 在项目管理环节的覆盖面，而不只是代码生成
- 让 AI 参与从需求到任务的转译与规范化
- 保持人工对 Sprint、优先级、资源调度的控制权
- 让 `Codex CLI` 只处理已经进入执行边界的任务
- 形成从需求分析到执行反馈的闭环

## 关键原则

- AI 可以自动创建 `Story` / `Task`
- 人工负责任务审核、Sprint 排期、资源分派
- `Codex CLI` 只从 Sprint 中读取 `Ready for Codex` 的任务
- `Codex CLI` 可以更新执行态状态，但不应接管项目治理字段
- 所有 AI 动作需要结构化输出与可审计回写

## 能力分层

### L1 需求理解

- 读取 Epic / 需求单
- 提炼目标、范围、约束、验收标准、风险

### L2 故事定义

- 从需求生成 Story
- 补齐用户价值、场景、AC、DoR、DoD

### L3 任务拆解

- 将 Story 拆解为 Task / Sub-task
- 区分后端、前端、测试、文档、发布等类型

### L4 设计辅助

- 结合代码仓库分析影响范围
- 生成技术实现建议、依赖与测试策略

### L5 Jira 写回

- 创建 / 更新 Epic、Story、Task、Comment、Label、相关字段

### L6 Sprint 执行

- 从 Sprint 拉取 `Ready for Codex` 的任务
- 调用 `Codex CLI` 执行
- 回写状态、摘要、测试结果、阻塞与风险
